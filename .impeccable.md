# Design Context

### Users
Developers building **AI agent UIs** — chat assistants, code copilots, tool-use streams, CLIs. They arrive on the landing wanting to (a) see the spinners working inside *real* agent screens, (b) browse all 55 to pick one, (c) copy the import + JSX snippet immediately.

### Brand Personality
Three words: **careful · platform · restrained**. The voice is closer to a tools-team docs site than a marketing splash — confident, sparse, no exclamation points. Emotion goal: *quiet competence*. Spinners are tiny — the page should feel tiny too.

### Aesthetic Direction
**Lynx-native muted.** Tinted neutrals (warm graphite ↔ warm paper). One signature accent in Lynx's brand red family (oklch around 0.62 / 0.22 / 25 — a tinted ember, not neon). No glowing accents. No purple-to-cyan gradients. No gradient text. No glassmorphism. No icon-above-every-heading. No cards-in-cards.

Light and dark themes, **system preference by default**. Light uses warm off-white (≈#FAF7F2 tinted toward the accent); dark uses warm graphite (≈#14110E). Mono is used only for code and frame strings — *never* as the body font.

References (in spirit, not visual): Stripe Docs (restraint), Linear Method (typography), Vercel design (clean spacing). Anti-references: AI-slop landing pages — dark+cyan+purple, neon hero metric, "Built with AI" badge.

### Design Principles
1. **The spinners are the hero — the chrome is silent.** Frame characters are typographically large enough to read; everything else recedes.
2. **Show, don't list.** Lead with *signature screens* (mock agent UIs running real `<lynx-view>` Lynx-for-Web bundles) before showing the catalogue.
3. **One accent. One typeface pair.** A serif display (e.g. Fraunces or similar editorial serif) + a calm sans (e.g. InterDisplay/Geist/Söhne family). Mono only for code blocks.
4. **Asymmetric, not centered.** Left-aligned headers, off-grid asides. Cards aren't the default container.
5. **Lynx port is an implementation detail.** The landing speaks to spinner users, not to engineers reviewing the port. The deck is a *footnote* at the bottom: "see how we ported it".
